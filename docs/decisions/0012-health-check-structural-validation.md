# 健康检查从字符长度阈值改为结构化校验

> 原决策 12（DESIGN.md v0.6 设计决策附录）

design-doc v0.4 的"description < 20 字符即警告"是机械阈值——21 字符的烂 description 同样会让 Agent 无法激活，而 18 字符的精炼 description 完全可能更好。决策弃用字数判定，改为按内容判断（DESIGN.md §1.5）：v0.1 用确定性规则检测是否包含触发场景动词；v0.2 起用 embedding 查同源描述是否语义重复、用 LLM 评估描述是否清晰且聚焦触发场景。字数不再作为质量判定条件（仅保留 spec 的 1024 字符上限作合规校验）。
