# 安全契约边界——"风险可见"而非"安全保证"

> 原决策 24（DESIGN.md v0.6 设计决策附录）

静态扫描无法证明任何脚本绝对安全，承诺"安全"是过度承诺。决策把 `scripts/` 安全预检的契约改成"风险可见 + 默认保守 + 高危项强制确认"（DESIGN.md §1.11）：明确列出能做的（symlink 不跟随、2MB 体积上限、二进制资产清单、高危规则扫描）和明确做不到的（SKILL.md 正文恶意指令、外部文件引用、高级混淆）。给用户**展示**风险，让用户**决策**是否接受，AgentMix **不替**用户判断安全与否——这是契约边界，写明白避免误期待。
