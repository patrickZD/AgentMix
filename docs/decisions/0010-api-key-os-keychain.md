# API 密钥统一通过 OS keychain，不静默降级

> 原决策 10（DESIGN.md v0.6 设计决策附录）

考虑过明文配置文件（直接淘汰，安全性不可接受）、加密文件 + 主密码（UX 摩擦 + KDF 选择长尾）、环境变量（GUI 应用读取不稳定）、云端托管（破"本地开源"定位）后，决定采用 OS keychain（`keyring` crate 跨三平台抽象）作为唯一密钥存储方式。诚实承认其安全边界：能防静态读取与误提交，**不能**防同用户进程读取——但已显著好于明文。Keychain 不可用时，依赖功能禁用并在 UI 明示原因，不允许静默降级到弱存储方案（DESIGN.md §4.7）。
